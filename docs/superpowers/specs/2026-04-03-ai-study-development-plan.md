# AI_Study 开发计划

## 项目阶段规划

| 阶段 | 名称 | 周期 | 主要目标 |
|------|------|------|----------|
| Phase 0 | 基础架构 | 1-2天 | 项目初始化、技术框架搭建 |
| Phase 1 | 核心功能 | 3-4天 | 用户系统、科目管理、题目生成 |
| Phase 2 | 学习流程 | 2-3天 | 闯关测验、即时阅卷、AI讲解 |
| Phase 3 | 游戏化 | 2-3天 | 成就系统、积分排行、打卡功能 |
| Phase 4 | 家长端 | 1-2天 | 报告查看、目标设置、奖励配置 |
| Phase 5 | 完善优化 | 2-3天 | 响应式适配、UI优化、Bug修复 |

**预计总工期：11-17天**

## 技术选型详细

### 前端
| 技术 | 用途 |
|------|------|
| Next.js 14 | React 框架 |
| Tailwind CSS | 样式框架 |
| Zustand | 状态管理 |
| React Query | 数据获取 |

### 后端
| 技术 | 用途 |
|------|------|
| Next.js API Routes | API 接口 |
| Prisma | ORM |
| better-sqlite3 | SQLite 驱动 |

### AI
| 技术 | 用途 |
|------|------|
| AI SDK | AI 接口封装 |
| OpenAI SDK | GPT 接口 |
| 支持的模型 | GLM/千问/Kimi/混元/MiniMax |

### 其他
| 技术 | 用途 |
|------|------|
| PDF.js | PDF 解析 |
| Mammoth | Word 解析 |
| Vercel/本地 | 部署 |

## 风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| AI API 不稳定 | 功能不可用 | 本地缓存 + 重试机制 |
| PDF 解析复杂 | 题目生成不准 | 简化解析 + 模板辅助 |
| 多用户性能 | 卡顿 | 索引优化 + 分页 |
| 数据库迁移 | 数据丢失 | 定期备份 + 迁移脚本 |
