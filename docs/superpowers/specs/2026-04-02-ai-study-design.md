# AI_Study 小学全科学习系统设计文档

## 1. 项目概述

### 1.1 项目名称
**AI_Study** - 小学生全科AI学习辅助系统

### 1.2 项目定位
AI学伴 + 游戏化学习型系统，通过AI动态生成测验题目，游戏化闯关机制，帮助小学生巩固学习成果。

### 1.3 核心价值
- AI 动态生成不同难度试题，支持基础题和奥赛扩展题
- 错题自动记录，AI 讲解分析，定期提醒巩固
- 游戏化学习机制，提升学习兴趣和动力
- 支持自定义科目，与年级绑定

## 3. 功能模块

### 3.1 科目与年级系统

#### 科目管理
| 类型 | 说明 |
|------|------|
| **预设科目** | 数学、语文、英语（必选） |
| **自定义科目** | 可添加科学、编程、美术、音乐等 |

#### 年级绑定
- 支持 1-6 年级
- 科目与年级关联，根据年级显示对应教材版本

#### 教材版本
| 科目 | 支持版本 |
|------|----------|
| 数学 | 人教版、北师大版、苏教版、沪教版 |
| 语文 | 人教版、北师大版、部编版、苏教版 |
| 英语 | 人教版(PEP)、外研社版、北师大版、沪教版 |
| 自定义 | 可自定义教材名称 |

#### 初始化流程
1. 选择年级（1-6年级）
2. 确认预设科目 + 添加自定义科目
3. 为每个科目选择教材版本
4. 完成设置，开始学习

### 3.2 课程系统
- **课程结构**：科目 → 年级 → 单元 → 章节
- **教材接入**：
  - 支持上传 PDF/Word 教材文档，AI 读取内容
  - 支持 AI 根据内置大纲自主生成题目

### 3.3 闯关测验系统

#### 题目生成
- AI 根据教材内容或知识大纲动态生成题目
- 支持基础题和奥赛扩展题两类难度
- 题目类型：单选题、多选题、填空题、解答题

#### 题目配置（三层机制）
| 层级 | 说明 |
|------|------|
| 全局默认 | 管理员设置每关默认题目数量和类型 |
| 闯关前调整 | 学生可选择题目数量（5/10/15/20题）和类型 |
| AI 智能推荐 | AI 根据章节内容和学生水平推荐题目组合 |

#### 难度设计
- **知识点关联难度递进**：基础概念 → 应用 → 综合 → 拓展
- AI 根据学生答题情况动态推荐难度

### 3.4 即时阅卷与得分
- 答题完成后自动批改
- 实时显示得分和正确率
- 答题模式支持"考试模式"（全屏专注）和"练习模式"（随时可做）

### 3.5 AI 讲解系统
- **错题讲解**：点击"不懂"按钮，AI 针对性讲解
- **知识点关联**：讲解后引导相关知识点复习讨论
- **对话交互**：学生可自由追问，AI 实时回答
- 支持上传截图提问

### 3.6 错题本系统
- 自动记录所有错题
- AI 分析错因（粗心/概念不清/难度过高）
- 定期提醒进行错题巩固测验
- 错题打卡日历，连续巩固追踪

### 3.7 游戏化系统

| 元素 | 说明 |
|------|------|
| 知识地图 | 可视化科目/章节进度，像游戏地图一样点亮 |
| 闯关模式 | 每个章节一个小关卡，通关解锁下一关 |
| 成就系统 | "首次全对"、"连续7天打卡"、"攻克难题"等徽章 |
| 积分系统 | 做题积累积分，可兑换成就 |
| 排行榜 | 局域网内可见，激励学习 |
| 连续打卡 | 积累能量值，解锁隐藏关卡 |

### 3.8 学习报告
- 知识点掌握度雷达图
- 学习曲线（正确率变化）
- 答题用时统计
- 成就解锁进度

### 3.9 家长端
- 查看孩子学习报告
- 设置每日/每周学习目标
- 配置奖励规则（积分兑换什么）
- 查看家庭排行榜
- 管理科目与年级设置

### 3.10 多用户系统
- 家庭多个孩子独立档案
- 家长可切换不同孩子账号
- 每个孩子独立学习进度、错题本、积分成就
- 每个孩子有独立的科目和教材版本设置

## 4. AI 模型配置

### 4.1 支持的模型
- 智谱 GLM（默认）
- 阿里千问
- 月之暗面 Kimi
- 腾讯混元（可选）
- MiniMax（可选）

### 4.2 配置方式
- 后台管理系统配置 AI 模型 API Key 和接口地址
- 支持多个模型切换使用
- 模型调用策略可配置（成本优先/效果优先）

## 5. 数据模型

### 5.1 核心实体
- **用户 (User)**：id, name, role, parent_id, grade
- **科目 (Subject)**：id, name, icon, is_preset, is_required
- **用户科目 (UserSubject)**：id, user_id, subject_id, grade, textbook_version
- **教材版本 (TextbookVersion)**：id, subject_id, name, publisher
- **课程 (Course)**：id, subject_id, grade, name
- **单元 (Unit)**：id, course_id, name, order
- **章节 (Chapter)**：id, unit_id, name, order, difficulty
- **题目 (Question)**：id, chapter_id, type, difficulty, content, answer, explanation, source
- **答题记录 (AnswerRecord)**：id, user_id, question_id, answer, is_correct, time_spent
- **错题本 (WrongQuestion)**：id, user_id, question_id, wrong_count, last_review, next_review
- **成就 (Achievement)**：id, user_id, type, unlocked_at
- **积分 (Point)**：id, user_id, points, source
- **打卡记录 (CheckIn)**：id, user_id, date, completed

### 5.2 数据库
- SQLite，本地文件存储
- 数据库文件位置：`data/ai_study.db`

## 7. 用户使用流程

```
1. 首次使用
   ├── 选择/创建账号
   ├── 初始化设置
   │   ├── 选择年级（1-6年级）
   │   ├── 确认预设科目（语数外）
   │   ├── 添加自定义科目（可选）
   │   └── 设置教材版本
   └── 完成设置

2. 日常学习
   ├── 登录系统 → 选择孩子账号
   ├── 查看首页仪表盘
   │   ├── 今日任务进度
   │   ├── 错题待巩固提醒
   │   └── 推荐闯关
   ├── 选择关卡闯关
   │   ├── 设置题目数量和类型
   │   ├── AI 推荐题目组合
   │   └── 选择模式（练习/考试）
   ├── 开始答题
   ├── 答题完成 → 即时阅卷 → 显示得分
   ├── 错题 → AI 讲解 → 讨论 → 重新挑战
   └── 完成关卡 → 获得积分/成就 → 解锁下一关

3. 错题巩固
   ├── 查看错题日历
   ├── 选择错题进行巩固
   └── AI 讲解 + 讨论

4. 家长管理
   ├── 切换孩子账号
   ├── 查看学习报告
   ├── 设置学习目标
   ├── 配置奖励规则
   └── 管理科目教材
```

## 8. 设计规范

### 8.1 UI 风格
- **风格名称**：Claymorphism（软陶风格）
- **视觉效果**：柔和阴影、3D 凸起/凹陷效果、圆润可爱

### 8.2 配色方案
| 用途 | 颜色代码 |
|------|----------|
| 主色 | #2563EB（学习蓝） |
| 辅色 | #3B82F6 |
| 强调色 | #F97316（活力橙） |
| 成功色 | #22C55E |
| 错误色 | #EF4444 |
| 背景色 | #F8FAFC |
| 文字色 | #1E293B |

### 8.3 字体规范
- **中文字体**：华文楷体（STKaiti）
- **英文/数字字体**：Nunito
- **基础字号**：18px（body）
- **标题字号**：20-24px

### 8.4 圆角规范
| 元素 | 圆角 |
|------|------|
| 大卡片 | 32px |
| 小卡片/按钮 | 20px |
| 输入框 | 12px |

### 8.5 阴影效果
```css
/* 凸起效果 */
box-shadow: 8px 8px 16px rgba(0,0,0,0.1), -8px -8px 16px rgba(255,255,255,0.8);

/* 凹陷效果 */
box-shadow: inset 4px 4px 8px rgba(0,0,0,0.1), inset -4px -4px 8px rgba(255,255,255,0.8);
```

### 8.6 响应式断点
| 设备 | 断点 | 布局 |
|------|------|------|

## 9. 设计原则

1. **简单易用**：小学生能独立操作，界面清晰直观
2. **AI 驱动**：AI 贯穿整个学习流程，而非简单工具
3. **游戏化适度**：游戏元素提升兴趣，但不喧宾夺主
4. **本地优先**：数据本地存储，保护隐私
5. **可扩展**：模块化设计，便于功能扩展
